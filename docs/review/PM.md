# PM 审阅与交付建议（ProxyPrice）

更新时间：2025-12-28

## 目标（MVP→生产级）

- 让用户**在 30 秒内**完成“选类型→看对比→点进某家→算成本→跳转官网”闭环。
- 保证每一个可见入口都**不 404、信息自洽、可解释、可追溯**（数据口径 + 更新时间 + 免责声明）。
- 用“透明、可复核的计算方法”建立信任：为什么这个 $/GB 可比、哪些不可比、不可比怎么展示。

## 当前亮点

- 静态站（Astro + Islands）方向正确：性能与部署成本优。
- 数据管道（CSV→JSON）+ pytest 覆盖了关键 parsing/normalization。
- 页面骨架完整：类型页、列表页、详情页、计算器页已具备。

## 风险与阻塞（必须优先解决）

1. **站内 404 / 断链风险**
   - Footer 指向 `/about`、`/methodology`，但页面不存在。
   - 默认 OpenGraph `og-image` 引用不存在（分享卡片会破图）。
2. **数据口径与“可比性”解释不足**
   - 首页宣称“Normalize all pricing models to $/GB”，但数据里多数记录不可比（per-ip/per-proxy/per-thread）。
   - 需要在 Methodology 明确：哪些可比、不可比如何呈现、排序依据是什么。
3. **部署配置噪音 / 误导**
   - Cloudflare 适配器会输出 KV Session binding 提示，但站点实际为静态站，不需要会话。

## 建议路线图（按价值/风险优先级）

### P0（本次必须交付）

- 站内必备页面：`About`、`Methodology`、`404`。
- SEO/分享卡片：提供可用的 `og-image`（并在 layout 中引用存在的资源）。
- “数据更新时间”统一来源（从生成的 JSON 读取），避免硬编码导致信息不一致。
- 详情页对“不可比”的 tier 展示做防呆：不再生成错位表格；用明确的 unit/模型展示，或降级为“仅提供链接 + 说明”。
- 移除/简化不必要的 Cloudflare SSR 适配器配置，保持“纯静态”部署路径清晰。

### P1（增强体验，推荐本周内做）

- 列表页增强：搜索/过滤（proxy type、是否可比、是否有 trial、价格区间）。
- “推荐逻辑”透明化：计算器展示“为什么推荐它”（选择了哪个 tier、是否按≥带宽选最便宜、是否支持 PAYG）。
- 内容与信任：增加“免责声明 / 数据来源 / 更新频率 / 纠错入口（提交 PR/表单）”。

### P2（增长与运营）

- Provider 页面内容模块化：优势/劣势、适用场景、FAQ、对比同类（内部链接提升 SEO）。
- 数据更新自动化（之后再做）：爬取/半自动校验 + CI 生成 PR。

## 交付验收（P0）

- 任意页面无控制台报错、无断链（至少站内链接 0 404）。
- `/about`、`/methodology` 可访问且解释清晰；`/404` 生效。
- 分享卡片（OG/Twitter）引用的图片资源真实存在且可被爬虫抓取。
- 数据更新时间在全站展示一致，且可由数据生成脚本自动更新。
